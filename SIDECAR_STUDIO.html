How To Use This Page

                  
This page is a portable editing workspace for the JavaScript connector or sidecar layer, not a replacement for the extension approval flow.

                

              

              

                

                  
Editor

                  
Write the script body, a multiline Markdown extended description, and an optional settings schema. Save it locally, then export either the JSON record or the assembled JavaScript bundle.

                

                

                  
Library

                  
Maintain multiple variants, import older exports, duplicate experiments, and hand off bundles without requiring the extension on the editing machine.

                

                

                  
Settings Schema

                  
The schema field is plain JSON. Keep it aligned with the declared bookmarklet settings model if the script will later be used with Bookmarklet Bridge.

                

                

                  
Generated Bundle

                  
The bundle preview is a readable self-contained object with `name`, `version`, multiline Markdown `extendedDescription`, `settings`, and `run` so it is easy to audit before export.

                

              

            


            

              

                

                  
Recommended Script Shape

                  
Use this object shape when you want a connector definition that can later be adapted into the extension helper model.

                

              

              
{
  name: "Example Connector",
  version: 1,
  extendedDescription: "Short explanation of what this script does.",
  settings: {
    includeSelection: {
      type: "boolean",
      label: "Include selection",
      description: "Append current selection to the outgoing payload.",
      default: true
    }
  },
  async run(bridge) {
    const settings = await bridge.getSettings();
    await bridge.toast("Running connector");
    return settings;
  }
}

            


            

              

                

                  
Useful Notes

                

              

              

                

                  
Import Format

                  
    
                    
  • A single script import can be one object with `id`, `name`, `version`, `extendedDescription`, `settingsText`, and `runBody`.
    • 
                    
  • A full workspace import can be `{ scripts: [...] }`.
    • 
                  

                

                

                  
Export Strategy

                  
    
                    
  • Use JSON when you want the editable record.
    • 
                    
  • Use JS when you want a readable handoff bundle.
    • 
                    
  • Keep versions explicit so diffs and approvals stay understandable later.
    •

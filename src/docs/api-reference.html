Bookmarklet Bridge API v2

Bookmarklet Bridge gives bookmarklets a small privileged API for cross-origin requests, page feedback, generated downloads, URL-based downloads, and clipboard writes.

What the helper hides

Registration, approval flow, execution IDs, request IDs, and the underlying window.postMessage transport.

What bookmarklet code should do

Read the page, prepare a small payload, call one or more bridge methods, and handle errors clearly.

Entry Point

Bookmarklets are expected to use the generated helper model:

runBookmarklet({
  name: "Example",
  version: 1,
  extendedDescription: `## What this does

- Fetches profile data
- Shows the result on the page

Only allow this if you trust the code below.`,
  settings: {
    includeSelection: {
      type: "boolean",
      label: "Include selection",
      description: "Append the current selection to the payload.",
      default: true
    }
  },
  async run(bridge) {
    const settings = await bridge.getSettings();
    await bridge.toast("Running...");
    const result = await bridge.get("https://example.com/api/me");
    console.log(result);
  }
});
Field Type Meaning
name string Stable human-readable bookmarklet identifier.
version number Integer version for the bookmarklet definition.
run async function Receives the bridge API after registration succeeds.
settings object Optional declared bookmarklet-scoped settings schema shown in approval and editable in the approved detail view.
extendedDescription string Optional Markdown shown in approval and policy detail views.

Actions

bridge.getSettings()

Read the current bookmarklet-scoped settings for this exact approved definition.

Success result: an object containing the validated current values merged with declared defaults.

bridge.get(url, options?)

Cross-origin GET request executed by the extension.

Parameter Type Notes
url string Must be a full URL.
options.headers record<string, string> Optional. Some forbidden headers are stripped by the bridge.

Success result: { status, headers, data } where data is parsed JSON when the response is JSON, otherwise text.

bridge.post(url, body, options?)

Cross-origin JSON POST request executed by the extension.

Parameter Type Notes
url string Must be a full URL.
body JSON-serializable value Serialized as JSON by the bridge.
options.headers record<string, string> Optional extra headers.

Success result: { status, headers, data }.

bridge.toast(message, options?)

Shows in-page feedback rendered by the extension content script.

Parameter Type Notes
message string Visible toast text.
options.variant "success" | "info" | "error" Optional. Defaults to info.
options.durationMs number Optional. Uses extension defaults if omitted.

Success result: { shown: true, payload: { message, variant, durationMs } }.

bridge.download({ filename, content?, bytesBase64?, mimeType? })

Saves generated text or base64-decoded binary content through the browser download manager.

Field Type Notes
filename string Required. Sanitized before the browser sees it.
content string Optional. Generated text content.
bytesBase64 string Optional. Base64-encoded binary bytes.
mimeType string Optional. Defaults to UTF-8 text.

Provide exactly one of content or bytesBase64. Success result: { downloadId, filename, mimeType, sizeBytes }.

bridge.downloadUrl({ url, filename? })

Starts a browser-managed download directly from a URL.

Field Type Notes
url string Required. Must be a full URL.
filename string Optional. Sanitized if provided.

Success result: { downloadId, url, filename? }. Allowed-origin settings still apply.

bridge.copyText(text)

Copies generated text through the extension clipboard permission.

Parameter Type Notes
text string Required. Text only. Clipboard content is not retained in logs.

Success result: { copied: true, sizeBytes }.

Registration And Approval

Before any action can run, the helper sends a registration message. Approval is tied to bookmarklet identity derived from:

  • name
  • version
  • readable source derived from the bookmarklet

If any of those change, the bridge treats it as a new bookmarklet definition and may require approval again.

{
  namespace: "bookmarklet-bridge",
  version: 2,
  kind: "register",
  requestId: "unique-string",
  executionId: "unique-per-run",
  bookmarklet: {
    name: "Example",
    version: 1,
    source: "async run(bridge) { ... }",
    extendedDescription: "## Summary\n\nShown in approval UI."
  }
}

The extended description is optional. It is rendered as Markdown in the allow / deny popup and in the management UI when inspecting approved or denied bookmarklets.

Responses

Every request receives exactly one response with the same requestId.

Success

{
  namespace: "bookmarklet-bridge",
  version: 2,
  requestId: "same-request-id",
  ok: true,
  result: { ... }
}

Error

{
  namespace: "bookmarklet-bridge",
  version: 2,
  requestId: "same-request-id",
  ok: false,
  error: {
    code: "invalid_request",
    message: "payload.url must be a valid full URL."
  }
}

Error Codes

Code Meaning
approval_requiredThe bookmarklet must be reviewed first.
approval_dismissedThe approval flow was dismissed.
deniedThe bookmarklet definition was denied.
invalid_requestThe message shape or payload is invalid.
unsupported_actionThe action name is not supported.
origin_not_allowedThe target origin is blocked by extension settings.
payload_too_largeThe request exceeded a bridge size limit.
network_errorThe network request failed.
timeoutThe network request timed out.
download_failedThe browser could not start the download.
clipboard_write_failedThe browser could not write to the clipboard.
bridge_internal_errorAn internal extension error occurred.

Limits And Notes

  • Bridge messages use namespace: "bookmarklet-bridge" and version: 2.
  • executionId stays stable within one bookmarklet run.
  • GET requests cannot include a request body.
  • POST bodies must be JSON-serializable.
  • Header count is capped.
  • Generated download content is capped at 1 MiB.
  • Clipboard text is capped at 256 KiB.
  • Logs intentionally avoid storing POST bodies, downloaded file contents, and clipboard text.

Full Example

runBookmarklet({
  name: "Page summary helper",
  version: 1,
  async run(bridge) {
    const summary = [
      "# " + (document.title || "Untitled"),
      "",
      location.href,
      "",
      window.getSelection ? String(window.getSelection()).trim() : ""
    ].filter(Boolean).join("\n");

    await bridge.download({
      filename: "page-summary.md",
      content: summary,
      mimeType: "text/markdown"
    });

    await bridge.downloadUrl({
      url: "https://example.com/files/reference.pdf",
      filename: "reference.pdf"
    });

    await bridge.copyText(summary);

    await bridge.toast("Summary copied and saved", {
      variant: "success",
      durationMs: 2200
    });
  }
});
